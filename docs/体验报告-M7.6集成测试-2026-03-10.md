## 三、关键问题

### 🔴 P0: Client 未向 LLM 传递 MCP Tools

**问题描述**：
Client 的 `ContextBuilder` 只构建消息列表（system + 历史消息 + 用户消息），但没有：
1. 从 Server 获取可用的 MCP Tools 列表
2. 将 Tools 传递给 LLM 请求

**影响**：
- LLM 不知道有哪些工具可用
- LLM 无法调用 `create_character`、`start_combat` 等工具
- 游戏体验依赖 LLM 纯文本生成，无法执行规则

**代码位置**：
- `packages/client/internal/service/context_builder.go:52-102`
- `packages/client/internal/service/chat.go:78-84`

**修复方案**：
```go
// 1. ContextBuilder 需要获取 Tools
func (b *ContextBuilder) GetAvailableTools(ctx context.Context) ([]llm.Tool, error) {
    // 从 Server 获取 MCP Tools 列表
    serverTools, err := b.serverClient.ListTools(ctx)
    if err != nil {
        return nil, err
    }

    // 转换为 LLM Tool 格式
    tools := make([]llm.Tool, len(serverTools))
    for i, t := range serverTools {
        tools[i] = llm.Tool{
            Type: "function",
            Function: llm.FunctionDef{
                Name:        t.Name,
                Description: t.Description,
                Parameters: t.InputSchema,
            },
        }
    }
    return tools, nil
}

// 2. ChatService.SendMessage 需要传递 Tools
func (s *ChatService) SendMessage(...) {
    // ... 构建 messages ...

    // 获取可用工具
    tools, err := s.contextBuilder.GetAvailableTools(ctx)
    if err != nil {
        return nil, fmt.Errorf("获取工具列表失败: %w", err)
    }

    // 调用 LLM 时传递 Tools
    llmResp, err := s.llmClient.Chat(ctx, &llm.ChatRequest{
        Messages:    messages,
        Tools:       tools,  // 传递工具列表
        Temperature: 0.7,
    })
    // ...
}
```

### 🔴 P0: Session/Campaign ID 同步问题

**问题描述**：
- Client 创建 Session 时生成自己的 UUID
- Server 创建 Campaign 时也生成自己的 UUID
- 两者无法同步

**影响**：
- Client 的 `session_id` 在 Server 上不存在
- 消息保存失败（外键约束）
- 需要手动同步 ID

**当前 workaround**：
```bash
# 1. 创建 Client Session
SESSION_RESP=$(curl -s -X POST http://localhost:8080/api/sessions ...)

# 2. 手动在 Server 创建 Campaign（使用相同 ID）
# 目前 Server 不支持自定义 ID，需要手动插入数据库
```

**修复方案**：

**方案 A：Client 先调用 Server 创建 Campaign**
```go
// packages/client/internal/service/session.go
func (s *SessionService) CreateSession(ctx context.Context, req *CreateSessionRequest) (*models.Session, error) {
    // 1. 先通过 Server 创建 Campaign
    campaign, err := s.serverClient.CreateCampaign(ctx, &server.CreateCampaignRequest{
        Name:        req.Name,
        DMID:        req.CreatorID,
        Description: req.Description,
    })
    if err != nil {
        return nil, fmt.Errorf("创建 Campaign 失败: %w", err)
    }

    // 2. 使用 Server 返回的 Campaign ID 作为 Session ID
    session := &models.Session{
        ID:           campaign.ID,  // 使用 Server 生成的 ID
        Name:         req.Name,
        // ...
    }
    // ...
}
```

**方案 B：Server create_campaign 支持自定义 ID**
```go
// packages/server/internal/api/tools/campaign.go
func (t *CampaignTools) CreateCampaignTool() mcp.Tool {
    return mcp.Tool{
        Name:        "create_campaign",
        Description: "...",
        InputSchema: map[string]any{
            // 添加可选的 id 字段
            "id": map[string]any{
                "type":        "string",
                "description": "可选的自定义 ID",
            },
            // ... 其他字段
        },
    }
}

func (t *CampaignTools) HandleCreateCampaign(...) {
    // 如果提供了 id，使用自定义 ID
    if id, ok := args["id"].(string); ok && id != "" {
        campaign.ID = id
    }
    // ...
}
```

## 四、架构问题

### 1. Client 和 Server 的职责边界不清

| 功能 | Client | Server | 问题 |
|------|--------|--------|------|
| Session 管理 | ✅ 有 | - | Session ≠ Campaign |
| Campaign 管理 | ❌ 无 | ✅ 有 | 需要同步 |
| 消息存储 | ✅ 有 | ✅ 有 | 重复存储 |
| 角色管理 | ❌ 无 | ✅ 有 | 只在 Server |
| 战斗管理 | ❌ 无 | ✅ 有 | 只在 Server |

**建议**：
- Client 应该是薄层，只负责 LLM 编排和用户交互
- Server 应该是唯一数据源
- Session 应该是 Campaign 的别名/引用，而不是独立实体

### 2. MCP Tool 调用流程不完整

**当前流程**（有问题）：
```
用户输入 -> Client Handler -> ChatService -> LLM (无 Tools) -> 纯文本响应
```

**期望流程**：
```
用户输入 -> Client Handler -> ChatService -> 获取 Tools -> LLM (带 Tools) -> Tool Call -> Server MCP -> 结果 -> LLM -> 最终响应
```
