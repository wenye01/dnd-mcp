## 二、协同需求分析

### 2.1 设计文档要求（来自 `docs/server/关键技术点讨论.md`）

根据已决策的 Server 设计：

```
用户输入 -> Client 获取上下文 -> LLM 决策 -> 调用 MCP Tool -> 规则引擎执行 -> 保存对话 -> 返回结果
```

**关键决策**:
1. **Server 统一存储对话历史** - Client 不存储（可选缓存）
2. **上下文压缩** - Server 提供简单滑动窗口（默认 20 条），Client 可自行高级压缩
3. **两种 Client 模式**:
   - 完整模式（我们的 Client）：获取原始数据，自行压缩
   - 简化模式（Claude Desktop 等）：使用 Server 压缩后的上下文

### 2.2 整局游戏流程分析

从创建战役到完整游玩需要的功能：

```
┌─────────────────────────────────────────────────────────────────────┐
```

## 四、详细重构方案

### 4.1 P0-1: 重构 ContextBuilder

**当前实现** (`packages/client/internal/service/context_builder.go`):
```go
func (b *ContextBuilder) BuildContext(ctx context.Context, sessionID, userMessage string) ([]llm.Message, error) {
    // 从本地 MessageStore 加载历史
    history, err := b.messageStore.List(ctx, sessionID, maxHistory)
    // ...
}
```

**目标实现**:
```go
func (b *ContextBuilder) BuildContext(ctx context.Context, campaignID, userMessage string) ([]llm.Message, error) {
    // 调用 Server get_context
    serverCtx, err := b.serverClient.GetContext(ctx, campaignID, b.messageLimit, b.includeCombat)
    // 转换为 LLM 消息格式
    // ...
}
```

**改动点**:
1. 注入 `ServerClient` 依赖
2. 移除 `messageStore` 和 `sessionStore` 依赖
3. 调用 `get_context` 或 `get_raw_context`
4. 转换 Server 返回的 Context 格式为 LLM Message 格式

### 4.2 P0-2: 重构 ChatService 消息保存

**当前实现** (`packages/client/internal/service/chat.go`):
```go
func (s *ChatService) SendMessage(...) (*models.Message, error) {
    // 保存用户消息到本地
    if err := s.messageStore.Create(ctx, userMessage); err != nil { ... }
    // ...
    // 保存助手消息到本地
    if err := s.messageStore.Create(ctx, assistantMessage); err != nil { ... }
}
```

**目标实现**:
```go
func (s *ChatService) SendMessage(...) (*models.Message, error) {
    // 保存用户消息到 Server
    if err := s.serverClient.SaveMessage(ctx, campaignID, userMsg); err != nil { ... }
    // ...
    // 保存助手消息到 Server
    if err := s.serverClient.SaveMessage(ctx, campaignID, assistantMsg); err != nil { ... }
}
```

**改动点**:
1. 注入 `ServerClient` 依赖
2. 移除 `messageStore` 依赖
3. 调用 `save_message` 保存消息

### 4.3 P0-3: 适配 Campaign 术语

**方案**: 保持 API 兼容，内部映射

```go
// 兼容层
type Session = Campaign  // 别名

// API 路由保持兼容
// /api/sessions/:id  -> 内部使用 campaignID
// /api/campaigns/:id -> 新路由
```

### 4.4 新增 ServerClient 接口

```go
// packages/client/internal/server/client.go
type ServerClient interface {
    // 上下文管理
    GetContext(ctx context.Context, campaignID string, limit int, includeCombat bool) (*Context, error)
    GetRawContext(ctx context.Context, campaignID string) (*RawContext, error)
    SaveMessage(ctx context.Context, campaignID string, msg *Message) error

    // MCP Tool 调用（已有接口）
    CallTool(ctx context.Context, toolName string, args map[string]any) (map[string]any, error)
}
```
