# 概述

## 1.1 目标

将 MCP Server 详细设计书转化为任务驱动的增量开发计划。每个任务是完整的垂直切片，可独立开发、测试、验证。

---

## 1.2 输入

- 设计文档: [详细设计.md](../详细设计.md)
- 需求规格: [高层次设计.md](../高层次设计.md)
- 变更文档: [FVTT导入需求补充.md](../FVTT导入需求补充.md)

---

## 1.3 约束

- **技术栈**: Go 1.24+, PostgreSQL
- **架构模式**: Handler → Service → Store → Models
- **测试要求**: 单元测试 + 集成测试
- **包路径**: `github.com/dnd-mcp/server`
- **图片存储**: 数据库 JSONB (Base64)
- **导入策略**: 核心字段转换 + 保留原始 JSON

---

## 1.4 任务结构

每个任务遵循垂直切片原则：

```
任务
├── 数据层 (Models)
│   └── 数据结构定义
├── 业务层 (Service)
│   └── 业务逻辑实现
├── 接口层 (Tools)
│   └── MCP Tool 暴露
└── 测试层 (Tests)
    ├── 单元测试
    └── 集成测试
```

---

## 1.5 开发流程

```
1. 阅读任务定义
   └── 理解需求来源和实现范围

2. 检查依赖
   └── 确认前置任务已完成

3. 实现任务
   ├── 数据层
   ├── 业务层
   ├── 接口层
   └── 测试层

4. 验证
   ├── 编译通过
   ├── 测试通过
   └── 代码审查

5. 提交
   └── 标记任务完成
```
