# 开发技能

严格按照开发计划和详细设计进行功能实现，完成后进行完整的编译和测试验证。

## 输入参数

**开发计划**: $ARGUMENTS

参数格式：
- `<plan-doc>` - 开发计划文档路径
- `<plan-doc> <task-id>` - 开发计划文档路径 + 具体任务ID（如 T1-1）

## 核心原则

```
1. 严格遵守设计 - 不添加设计之外的功能
2. 垂直切片实现 - 数据层 → 业务层 → 接口层 → 测试层
3. 干净环境验证 - 确保在全新环境可复现
4. 错误优先修复 - 遇到问题先尝试自动修复
5. 提交需人工确认 - 不自动提交，只提示
```

## 执行流程

### 第一步：加载计划和设计

```
1. 读取开发计划文档
2. 确认任务ID（如未指定，列出可用任务供选择）
3. 读取对应的详细设计文档
4. 确认任务的前置依赖已完成
```

### 第二步：任务分析

```
对目标任务进行确认：

1. 需求来源
   - 设计文档章节
   - 需求ID

2. 实现范围确认
   ├─ 数据层: 模型定义、存储实现
   ├─ 业务层: 服务逻辑、业务规则
   ├─ 接口层: HTTP Handler、路由注册
   └─ 测试层: 单元测试 + 集成测试

3. 文件清单确认
   - 列出所有需要新增/修改的文件
   - 确认目录结构

4. 约束条件
   - 前置任务
   - 技术约束
   - 边界条件
```

### 第三步：实现代码

```
严格按照设计文档的顺序实现：

1. 数据层
   - 定义模型结构体
   - 实现存储接口
   - 添加必要的数据库操作

2. 业务层
   - 实现服务方法
   - 添加业务规则验证
   - 处理错误情况

3. 接口层
   - 实现 HTTP Handler
   - 注册路由
   - 定义请求/响应 DTO

4. 测试层
   - 编写单元测试
   - 编写集成测试

实现过程中：
- 严格遵循设计文档，不添加额外功能
- 使用项目中已有的模式和工具
- 遵循代码规范（见 CLAUDE.md）
- 复用现有代码，避免重复
```

### 第四步：干净环境编译

```
清理环境并重新编译：

1. 清理构建产物
   PowerShell:
   Remove-Item -Recurse -Force bin/ -ErrorAction SilentlyContinue
   go clean -cache -testcache

2. 清理 Go 模块缓存（可选，更彻底）
   go clean -modcache

3. 重新下载依赖
   go mod download
   go mod tidy

4. 编译项目
   go build ./...

5. 检查编译结果
   - 成功 → 进入测试阶段
   - 失败 → 进入错误修复流程
```

### 第五步：干净环境测试

```
重置测试环境并运行测试：

1. 重置依赖服务
   - 清空测试用 Redis 数据库（DB 1）
   - 重置测试数据库

2. 终止残留进程
   - 检查并终止占用端口的进程

3. 运行测试套件
   按顺序执行：
   a. 单元测试
      go test -v ./tests/unit/... -cover

   b. 集成测试
      go test -v ./tests/integration/... -cover -timeout 30s

   c. 完整测试（可选）
      .\scripts\test-all.ps1

4. 检查测试结果
   - 全部通过 → 进入提交提示阶段
   - 有失败 → 进入错误修复流程
```

### 第六步：错误修复流程

```
当编译或测试失败时：

1. 分析错误
   - 读取错误信息
   - 定位错误位置
   - 理解错误原因

2. 尝试修复
   - 最多尝试 3 次修复
   - 每次修复后重新编译/测试

3. 修复策略
   a. 编译错误：
      - 检查语法错误
      - 检查类型不匹配
      - 检查缺失的导入
      - 检查接口实现完整性

   b. 测试错误：
      - 检查测试数据
      - 检查断言条件
      - 检查模拟对象
      - 检查环境配置

4. 修复失败处理
   如果 3 次修复后仍然失败：

   ## ⚠️ 无法自动修复

   ### 错误类型
   [编译错误/测试错误]

   ### 错误信息
   ```
   [完整的错误输出]
   ```

   ### 已尝试的修复
   1. [修复尝试1]
   2. [修复尝试2]
   3. [修复尝试3]

   ### 建议的人工干预
   - [ ] 检查设计文档是否有遗漏
   - [ ] 检查依赖服务是否正常运行
   - [ ] 检查环境变量是否正确设置
   - [ ] 检查是否有权限问题

   ### 下一步
   请人工检查上述问题，或提供更多信息以继续调试。

   [停止执行，等待用户指示]
```

## 项目特定配置

针对此 Go 项目：

```
环境清理命令 (PowerShell):
  # 清理构建产物
  Remove-Item -Recurse -Force bin/ -ErrorAction SilentlyContinue

  # 清理 Go 缓存
  go clean -cache -testcache

  # 重置 Redis 测试数据库
  redis-cli -n 1 FLUSHDB

  # 终止占用端口的进程
  Get-NetTCPConnection -LocalPort 8080 -ErrorAction SilentlyContinue |
    Select-Object -ExpandProperty OwningProcess |
    ForEach-Object { Stop-Process -Id $_ -Force -ErrorAction SilentlyContinue }

构建命令:
  go build -o bin/dnd-api.exe ./cmd/api

测试命令:
  go test -v ./tests/unit/... -cover
  go test -v ./tests/integration/... -cover -timeout 30s
  .\scripts\test-all.ps1

代码检查:
  go vet ./...
  gofmt -l .
```
