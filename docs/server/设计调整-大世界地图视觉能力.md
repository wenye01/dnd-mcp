## 1. 需求描述

### 1.1 用户需求

> 地图的设计需要调整，当前使用了双地图设计，战斗地图不用调整，对于大世界地图，当前可能输入只是一张纯粹的图片，能力如同 https://www.dndhub.top/guide 这里的，为此可能需要添加一个 VLLM，或者视觉 mcp，llm 作为 dm 可以知道当前玩家位置，以及世界有什么地点，从而规划后续剧情

### 1.2 需求分析

- **类型**: 扩展（新增视觉能力）
- **目标组件**: Server
- **核心诉求**:
  1. **支持纯图片输入**: 大世界地图可以只是一张纯粹的图片，不需要预定义的格子系统
  2. **视觉理解能力**: 系统需要理解地图图片内容，识别地点、地形等信息
  3. **DM 剧情规划支持**: LLM 作为 DM 能够感知玩家位置和世界地点，用于规划后续剧情
  4. **保留战斗地图设计**: 战斗地图的格子系统设计保持不变

### 1.3 参考能力（D&D Hub）

- 可缩放、可拖拽的世界地图
- 点击添加标记（城镇、遗迹、营地等）
- 筛选标记（按类型）
- 情报手记（Wiki）记录人物、地点、线索等
- 线索板可视化推理
- 时间轴

## 3. 调整方案

### 3.1 系统架构视角 (Architect-System)

#### 3.1.1 架构影响

**现有架构**:
```
大地图（Grid 系统）
    ├── Grid.Cells[][] - 格子数据
    ├── Locations[] - 地点标记
    └── 基于格子计算移动
```

**调整后架构**:
```
大地图（双模式）
    ├── 模式1: Grid 系统（保留）
    │   ├── Grid.Cells[][]
    │   ├── Locations[]
    │   └── 基于格子计算移动
    │
    └── 模式2: 图片地图（新增）⭐
        ├── ImageData - 地图图片
        ├── VisualLocations[] - 视觉识别的地点
        ├── PlayerMarker - 玩家标记位置
        └── 通过视觉 MCP 识别内容
```

#### 3.1.2 技术方案

**最终方案: Client LLM 直接调用视觉 MCP** (已确认)

```
┌─────────────────────────────────────────────────────────────┐
│                      Client (MCP Client)                     │
├─────────────────────────────────────────────────────────────┤
│  LLM (DM)                                                    │
│      │                                                       │
│      ├── 1. 调用视觉 MCP 分析地图图片                         │
│      │       │                                               │
│      │       ▼                                               │
│      │   ┌─────────────┐                                     │
│      │   │  视觉 MCP   │ (外部服务)                          │
│      │   └─────────────┘                                     │
│      │       │                                               │
│      │       ▼                                               │
│      │   返回: 识别的地点列表、地形描述                        │
│      │                                                       │
│      ├── 2. DM 确认/调整地点                                  │
│      │                                                       │
│      └── 3. 调用 Server MCP Tools 提交数据                   │
│              │                                               │
│              ▼                                               │
└──────────────┼──────────────────────────────────────────────┘
               │
               ▼
┌─────────────────────────────────────────────────────────────┐
│                      MCP Server                              │
├─────────────────────────────────────────────────────────────┤
│  MapService                                                  │
│      │                                                       │
│      ├── create_visual_location (新增)                       │
│      │       └── 存储视觉识别的地点                           │
│      │                                                       │
│      ├── update_location                                     │
│      │       └── 更新地点信息                                 │
│      │                                                       │
│      ├── get_world_map                                       │
│      │       └── 返回地图数据 + PlayerMarker + Locations      │
│      │                                                       │
│      └── move_to (Image 模式)                                │
│              └── 更新 PlayerMarker 位置                       │
│                                                              │
│  职责:                                                       │
│  - 存储和管理地图数据                                         │
│  - 存储和管理地点数据                                         │
│  - 跟踪玩家位置 (PlayerMarker)                               │
│  - 不负责视觉分析                                             │
└─────────────────────────────────────────────────────────────┘
```

**优点**:
- Server 职责清晰，只负责数据存储和游戏状态管理
- Server 不需要集成 MCP 客户端
- 视觉能力由 Client 灵活选择和配置
- Server 实现更简单

**Server 需要提供的接口**:
1. `create_visual_location` - 创建视觉地点
2. `update_location` - 更新地点信息
3. `get_world_map` - 获取地图（支持 Image 模式）
4. `move_to` - 移动（支持 Image 模式）

**Client/LLM 负责的**:
1. 调用视觉 MCP 分析地图图片
2. 处理分析结果
3. DM 确认/调整地点
4. 调用 Server 接口存储数据

#### 3.1.4 兼容性考量

- **向后兼容**: 现有 Grid 模式地图完全保留，不影响已有设计
- **渐进增强**: 图片地图作为可选增强功能
- **降级策略**: 视觉 MCP 不可用时，回退到手动标记地点

#### 3.2.1 模型变更

**扩展 Map 模型**:

```go
// Map 地图实体
type Map struct {
    ID          string       `json:"id"`
    CampaignID  string       `json:"campaign_id"`
    Name        string       `json:"name"`
    Type        MapType      `json:"type"`         // world / battle

    // === 新增: 地图模式 ===
    Mode        MapMode      `json:"mode"`         // grid / image

    // Grid 模式字段（保留）
    Grid        *Grid        `json:"grid,omitempty"`

    // Image 模式字段（新增）⭐
    Image       *MapImage    `json:"image,omitempty"`

    // 通用字段
    Locations   []Location   `json:"locations"`    // 地点标记
    Tokens      []Token      `json:"tokens"`       // Token列表（战斗地图）
    ParentID    string       `json:"parent_id"`
    CreatedAt   time.Time    `json:"created_at"`
    UpdatedAt   time.Time    `json:"updated_at"`
}

// MapMode 地图模式（新增）
type MapMode string

const (
    MapModeGrid  MapMode = "grid"   // 格子模式（默认）
    MapModeImage MapMode = "image"  // 图片模式
)

// MapImage 地图图片（新增）⭐
type MapImage struct {
    URL          string `json:"url"`           // 图片 URL 或 Base64
    Width        int    `json:"width"`         // 图片宽度（像素）
    Height       int    `json:"height"`        // 图片高度（像素）
    Format       string `json:"format"`        // 格式：png, jpg, webp

    // 视觉分析结果（可选，首次分析后缓存）
    Analysis     *VisualAnalysis `json:"analysis,omitempty"`
    AnalyzedAt   *time.Time      `json:"analyzed_at,omitempty"`
}

// VisualAnalysis 视觉分析结果（新增）⭐
type VisualAnalysis struct {
    Summary      string           `json:"summary"`       // 地图整体描述
    Locations    []VisualLocation `json:"locations"`     // 识别的地点
    Terrains     []TerrainArea    `json:"terrains"`      // 识别的地形区域
    Landmarks    []Landmark       `json:"landmarks"`     // 标志性建筑/物体
}

// VisualLocation 视觉识别的地点（新增）⭐
type VisualLocation struct {
    ID          string  `json:"id"`
    Name        string  `json:"name"`           // 视觉识别的名称
    Description string  `json:"description"`    // AI 生成的描述
    Type        string  `json:"type"`           // town, dungeon, forest, mountain, etc.

    // 图片坐标（像素）
    PositionX   float64 `json:"position_x"`     // X 坐标（0-1 归一化）
    PositionY   float64 `json:"position_y"`     // Y 坐标（0-1 归一化）

    // 关联数据
    BattleMapID string  `json:"battle_map_id,omitempty"` // 关联的战斗地图
    CustomName  string  `json:"custom_name,omitempty"`   // DM 自定义名称
    IsConfirmed bool    `json:"is_confirmed"`            // DM 是否已确认
}

// TerrainArea 地形区域（新增）
type TerrainArea struct {
    Type        string  `json:"type"`           // forest, desert, water, etc.
    Description string  `json:"description"`
    Bounds      Rect    `json:"bounds"`         // 区域边界
}

// Landmark 标志物（新增）
type Landmark struct {
    Name        string  `json:"name"`
    Description string  `json:"description"`
    PositionX   float64 `json:"position_x"`
    PositionY   float64 `json:"position_y"`
}

// Rect 矩形区域
type Rect struct {
    X      float64 `json:"x"`
    Y      float64 `json:"y"`
    Width  float64 `json:"width"`
    Height float64 `json:"height"`
}
```

#### 3.2.2 业务规则调整

1. **地图模式选择**
   - 创建地图时，根据是否有 Image 数据自动选择模式
   - 可显式指定 Mode

2. **位置表示**
   - Grid 模式: 使用 `{x, y}` 格子坐标
   - Image 模式: 使用 `{position_x, position_y}` 归一化坐标 (0-1)

3. **移动规则**
   - Grid 模式: 基于格子计算距离和时间
   - Image 模式: 基于视觉分析的地点跳转

4. **视觉分析缓存**
   - 首次请求时进行视觉分析
   - 结果存储在 MapImage.Analysis
   - DM 可手动触发重新分析
