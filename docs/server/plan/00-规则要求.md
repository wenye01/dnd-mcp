## 0.2 实现原则

1. **规则优先**: 涉及游戏规则的代码实现（如计算AC、伤害、豁免等），必须先查阅规则书确认规则细节
2. **禁止猜测**: 不确定的规则点，必须查阅规则书或向用户确认，严禁凭印象或猜测实现
3. **模糊点人工介入**: 当规则书描述模糊或存在歧义时，**必须暂停并询问用户**如何处理，提供：
   - 规则书的原文描述
   - 可能的解释选项
   - 每种选项的影响说明
4. **规则引用**: 在代码注释中标注规则来源（如 `// PHB 第9章: Damage and Healing`）

## 0.5 外部格式类型规范

**背景**: BUG-M6.5-003 发现 FVTT/UVTT 导入时存在类型不兼容问题，因为 JSON 数字默认解析为 `float64`。

### 类型规范（强制）

| 字段类型 | Go 类型 | 示例 |
|----------|---------|------|
| 坐标/位置 (x, y) | `float64` | `1234.56`, `3971.63` |
| 尺寸/大小 (width, height) | `float64` | `637.5`, `425.25` |
| 角度/旋转 | `float64` | `45.0`, `90.5` |
| 百分比/比例 | `float64` | `0.85`, `0.5` |
| 距离/范围 | `float64` | `15.75`, `7.875` |
| 数量/计数（非负整数） | `int` | `3`, `10` |

### 实现要求

1. **所有坐标/尺寸/角度字段**: 使用 `float64`
2. **所有百分比/比例字段**: 使用 `float64`
3. **所有数量/计数字段（非负整数）**: 使用 `int`
4. **不确定时优先使用 `float64`**

### 辅助方法

对于需要整数值的场景，提供辅助方法：

```go
// GetXInt returns the X coordinate as an integer
func (d *FVTTDrawing) GetXInt() int {
    return int(d.X)
}
```

### 根因分析

**问题**: BUG-M6.5-003 修复不完整，同一类问题反复出现。

**根因**:
1. 测试数据不够全面（仅使用简单 sample_scene.json）
2. 类型定义没有完全遵循 FVTT 实际格式
3. 缺少端到端真实数据测试流程
4. 修复时只修复了当前报错的字段，没有系统性检查所有类似字段

**预防**:
1. 使用真实数据进行全面测试
2. 遵循本文档的类型规范
3. 修复时系统性检查所有同类字段
