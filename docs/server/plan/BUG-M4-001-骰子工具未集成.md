# BUG-M4-001: 骰子工具未集成

> **状态**: ✅ 已修复
> **优先级**: 高
> **关联里程碑**: [M4-骰子系统](M4-骰子系统.md)
> **发现日期**: 2026-02-27
> **修复日期**: 2026-02-27

---

## 问题描述

### 现象

M4 里程碑标记为已完成，但骰子工具未注册到 MCP Server：

```bash
$ curl -s http://localhost:8080/mcp/tools
{"tools":[]}

$ curl -s -X POST http://localhost:8080/mcp/tools/call \
  -H "Content-Type: application/json" \
  -d '{"name": "roll_dice", "arguments": {"formula": "1d20+5"}}'
{"content":[{"type":"text","text":"unknown tool: roll_dice"}],"isError":true}
```

### 影响

- M4 的 3 个骰子工具 (`roll_dice`, `roll_check`, `roll_save`) 无法通过 API 使用
- 里程碑验收标准"所有 3 个骰子 Tools 实现并通过测试"未完全达成
- 用户无法使用骰子功能

---

缺少：

## 修复方案

### 任务清单

| ID | 任务 | 复杂度 | 说明 |
|----|------|--------|------|
| FIX-1 | 在 main.go 中注册骰子工具 | S | 创建 DiceService 并注册 DiceTools |
| FIX-2 | 添加 E2E 测试验证工具可用 | S | 验证工具列表包含骰子工具 |

### 详细说明

#### FIX-1: 在 main.go 中注册骰子工具

**文件**: `packages/server/cmd/server/main.go`

**修改内容**:

```go
import (
    "github.com/dnd-mcp/server/internal/api/tools"
    "github.com/dnd-mcp/server/internal/service"
)

// 在 Step 5 位置添加:

// Step 5: Initialize services
diceService := service.NewDiceService()

// Step 6: Register Tools
diceTools := tools.NewDiceTools(diceService)
diceTools.Register(server.Registry())
```

#### FIX-2: 添加 E2E 测试验证工具可用

**新增/修改文件**: `packages/server/tests/e2e/tools_test.go`

**测试内容**:
1. 调用 `GET /mcp/tools` 验证工具列表
2. 验证包含 `roll_dice`, `roll_check`, `roll_save`
3. 调用 `roll_dice` 验证功能正常
