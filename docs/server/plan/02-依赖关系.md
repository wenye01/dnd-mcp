## 里程碑依赖说明

| 里程碑 | 依赖 | 说明 |
|--------|------|------|
| M1 | 无 | 基础设施，其他所有里程碑依赖 |
| M2 | M1 | 需要 MCP 框架和数据库 |
| M3 | M2 | 需要战役和游戏状态 |
| M4 | M3 | 需要角色属性 |
| M5 | M4 | 需要骰子系统 |
| M6 | M4 | 需要骰子（移动检定） |
| M7 | M2 | 需要战役上下文 |
| M8 | M4 (可选) | 可独立开发，但与骰子配合更好 |
| M9 | M3, M6 | 需要角色和地图模型 |

---

## 关键路径

```
M1 → M2 → M3 → M4 → M5 (战斗系统完成)
                 ↘
                   M6 → M9 (导入功能完成)
```

**最短开发路径**: M1 → M2 → M3 → M4 → M5 → M6 → M9

---

## 并行开发建议

完成 M4 后可并行开发：
- **路径 A**: M5 (战斗系统)
- **路径 B**: M6 (地图系统) + M9 (导入功能)

完成 M2 后可并行开发：
- **路径 C**: M3 → M4 (核心链)
- **路径 D**: M7 (上下文管理，独立)
