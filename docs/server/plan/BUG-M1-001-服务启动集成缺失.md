# BUG-M1-001: 服务启动集成缺失

> **状态**: ✅ 已修复
> **优先级**: 高
> **关联里程碑**: [M1-项目基础设施](M1-项目基础设施.md)
> **发现日期**: 2026-02-27
> **修复日期**: 2026-02-27

---

## 问题描述

### 现象

`packages/server/cmd/server/main.go` 只有占位符代码，未实现真正的服务启动逻辑：

```go
func main() {
    fmt.Println("DND MCP Server starting...")
    // TODO: Initialize configuration
    // TODO: Initialize database connection
    // TODO: Initialize MCP server
    // TODO: Start HTTP server
    fmt.Println("Server initialized successfully (placeholder)")
    os.Exit(0)
}
```

服务无法真正启动运行。

### 影响

- Server 无法部署提供实际服务
- M1 里程碑的验收标准未完全达成
- 后续业务功能无法通过真实服务验证

---

## 根因分析

### 开发计划缺口

M1 计划的验收标准只有：
- [x] 项目可编译，无错误
- [x] 数据库连接成功
- [x] MCP 协议框架可响应基本请求
- [x] 单元测试通过

缺少：
- 服务启动验证
- main.go 初始化逻辑测试

### 测试策略缺陷

集成测试 `tests/integration/mcp/server_test.go` 使用 `setupRouter()` 手动构建路由：

```go
func setupRouter(cfg *config.Config, server *mcp.Server) http.Handler {
    router := gin.New()
    // 手动重新实现所有路由...
}
```

而非使用 `mcp.Server.Start()` 中的真实路由。

**结果**：即使 `mcp.Server.Start()` 有问题，测试也能通过。

---

## 修复方案

### 任务清单

| ID | 任务 | 复杂度 | 说明 |
|----|------|--------|------|
| FIX-1 | 实现 main.go 服务启动逻辑 | M | 初始化配置、数据库、MCP Server 并启动 |
| FIX-2 | 修复集成测试 | S | 使用真实服务路由而非手动构建 |
| FIX-3 | 添加服务启动验证测试 | M | 验证服务能启动并响应请求 |

### 详细说明

#### FIX-1: 实现 main.go 服务启动逻辑

**文件**: `packages/server/cmd/server/main.go`

**实现内容**:
1. 加载配置 (`pkg/config.Load()`)
2. 初始化日志 (`pkg/logger`)
3. 连接数据库 (`internal/store/postgres`)
4. 执行数据库迁移
5. 创建 MCP Server 实例
6. 注册已实现的 Tools
7. 启动 HTTP 服务
8. 处理优雅关闭

#### FIX-2: 修复集成测试

**文件**: `packages/server/tests/integration/mcp/server_test.go`

**修改内容**:
- 删除 `setupRouter()` 函数
- 使用 `mcp.Server.Start()` 在随机端口启动真实服务
- 或使用 `httptest.Server` 配合 `mcp.Server.registerRoutes()`

#### FIX-3: 添加服务启动验证测试

**新增文件**: `packages/server/tests/e2e/server_startup_test.go`

**测试内容**:
1. 构建二进制文件
2. 启动服务进程
3. 调用 `/health` 端点验证
4. 调用 `/mcp/tools` 验证 Tool 注册
5. 清理进程
