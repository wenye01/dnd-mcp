# Milestone 9: 导入功能（角色/物品）

> **状态**: ⏳ 待开发
> **依赖**: [M3](M3-角色管理.md), [M6.5](M6.5-地图导入功能.md)
> **任务数**: 5
> **涉及 D&D 5e 规则**: 否
> **📅 更新日期**: 2026-03-01
> **📝 变更说明**: 地图导入功能移至 M6.5，本里程碑仅保留角色和物品导入

---

## 9.1 目标

实现 FVTT 角色和物品的导入功能，支持从 FVTT Actor 数据创建角色和物品。

---

## 9.2 范围

- **包含**: FVTT Actor 解析、Character 转换、Item 转换、import_character Tool、import_items Tool
- **不包含**:
  - 地图导入（已移至 M6.5）
  - UVTT 解析（已移至 M6.5）
  - LevelDB 解析（已移至 M6.5）
  - FVTT 导出
  - 光源系统
  - ActiveEffect
  - 非 dnd5e 游戏系统

---

## 详细任务定义

### Task T9-1: FVTT Actor Parser

**一句话描述**: 实现 FVTT Actor 格式解析器。

#### 需求来源

- FVTT 变更: [FVTT导入需求补充.md](../FVTT导入需求补充.md) - 第9.1节

#### 实现范围

**数据层**:
- 模型:
  - `FVTTActor` - Actor 根结构
  - `FVTTAbilities` - 六大属性
  - `FVTTAttributes` - HP/AC/速度等
  - `FVTTDetails` - 种族/职业/背景等
  - `FVTTSkills` - 技能数据
  - `FVTTTraits` - 特性/抗性等
  - `FVTTCurrency` - 货币
- 存储: 无

**业务层**:
- 服务: `FVTTActorParser.CanParse()`, `FVTTActorParser.Parse()`
- 规则: _id, system.abilities 字段检测

**接口层**:
- 端点: 无

**测试层**:
- 单元测试: 解析测试
- 集成测试: 真实 FVTT Actor 文件解析

#### 约束条件

- 前置任务: M6.5 (Import 模块框架)
- 技术约束: FVTT dnd5e system 格式
- 边界条件: 无效格式、非 dnd5e 系统

#### 验收标准

- [ ] 实现完成: FVTT Actor 解析器
- [ ] 测试通过: 真实 FVTT Actor 文件解析正确

#### 文件清单

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 新增 | packages/server/internal/import/format/fvtt_actor.go | FVTT Actor 格式定义 |
| 新增 | packages/server/internal/import/parser/fvtt_actor_parser.go | FVTT Actor 解析器 |
| 新增 | packages/server/tests/unit/import/fvtt_actor_parser_test.go | 单元测试 |
| 新增 | packages/server/tests/testdata/sample_actor.json | 测试数据 |
