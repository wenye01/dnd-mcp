> **状态**: 🔴 P0 阻塞

---

## 问题描述

### 现象

Client 调用 Server 保存消息时返回 404 错误：
```
保存消息失败 (status 404): 404 page not found
```

### 根本原因

**协议不匹配**：
- **Client HTTPClient** 使用 REST API 风格端点
- **Server** 只提供 MCP 协议 API

| 方法 | Client 调用 | Server 端点 |
|------|-------------|-------------|
| GetContext | `GET /api/campaigns/{id}/context` | ❌ 不存在 |
| SaveMessage | `POST /api/campaigns/{id}/messages` | ❌ 不存在 |
| CallTool | `POST /api/campaigns/{id}/tools/{name}` | ❌ 不存在 |
| 正确调用 | `POST /mcp/tools/call` | ✅ 存在 |

---

## 修复方案

### 架构决策

**Client 使用 MCP 协议与 Server 对接**

理由：
1. MCP 是 Server 的原生协议
2. 所有 Server 功能已通过 MCP Tool 暴露
3. 修改范围小，仅限 Client

### 端点映射

| 原方法 | 新实现 |
|--------|--------|
| `GetContext` | `POST /mcp/tools/call` with `name: "get_context"` |
| `GetRawContext` | `POST /mcp/tools/call` with `name: "get_raw_context"` |
| `SaveMessage` | `POST /mcp/tools/call` with `name: "save_message"` |
| `CallTool` | `POST /mcp/tools/call` with `name: args.toolName` |

### MCP Tool 映射

Server 已实现的 MCP Tools：

| Tool Name | 用途 |
|-----------|------|
| `get_context` | 获取压缩上下文 |
| `get_raw_context` | 获取原始上下文 |
| `save_message` | 保存消息 |
| `create_campaign` | 创建战役 |
| `create_character` | 创建角色 |
| ... | ... |

### Task T7.6-1: 重构 HTTPClient 使用 MCP 协议

**一句话描述**: 修改 `HTTPClient` 所有方法使用 `POST /mcp/tools/call` 端点。

#### 修改文件

| 文件 | 修改内容 |
|------|----------|
| `packages/client/internal/server/http.go` | 重构所有方法使用 MCP 协议 |

#### 实现要点

```go
// mcpCall 是核心方法，所有操作都通过它调用
func (c *HTTPClient) mcpCall(ctx context.Context, toolName string, args map[string]any) (*CallToolResponse, error) {
    url := fmt.Sprintf("%s/mcp/tools/call", c.baseURL)

    reqBody := map[string]any{
        "name":      toolName,
        "arguments": args,
    }

    body, _ := json.Marshal(reqBody)
    req, _ := http.NewRequestWithContext(ctx, "POST", url, bytes.NewReader(body))
    req.Header.Set("Content-Type", "application/json")

    resp, err := c.httpClient.Do(req)
    // ... 处理响应
}

// GetContext 调用 get_context tool
func (c *HTTPClient) GetContext(ctx context.Context, campaignID string, limit int, includeCombat bool) (*Context, error) {
    args := map[string]any{
        "campaign_id":    campaignID,
        "limit":          limit,
        "include_combat": includeCombat,
    }
    resp, err := c.mcpCall(ctx, "get_context", args)
    // ... 解析响应
}

// SaveMessage 调用 save_message tool
func (c *HTTPClient) SaveMessage(ctx context.Context, campaignID string, msg *Message) error {
    args := map[string]any{
        "campaign_id": campaignID,
        "role":        msg.Role,
        "content":     msg.Content,
        "player_id":   msg.PlayerID,
    }
    _, err := c.mcpCall(ctx, "save_message", args)
    return err
}

// CallTool 调用任意 tool
func (c *HTTPClient) CallTool(ctx context.Context, campaignID, toolName string, args map[string]any) (map[string]any, error) {
    // 确保 campaign_id 在参数中
    if _, ok := args["campaign_id"]; !ok {
        args["campaign_id"] = campaignID
    }
    resp, err := c.mcpCall(ctx, toolName, args)
    // ... 解析响应
}
```

#### 验收标准

- [ ] 所有方法使用 `POST /mcp/tools/call`
- [ ] 正确构造 MCP 请求格式
- [ ] 正确解析 MCP 响应格式
- [ ] 单元测试通过
