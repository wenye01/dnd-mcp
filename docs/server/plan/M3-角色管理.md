### Task T3-7: 死亡豁免规则

**一句话描述**: 实现 HP=0 时的昏迷/死亡判断规则，符合 PHB 第9章规范。

#### 需求来源

- **规则书**: PHB 第9章 - Damage and Healing / Dropping to 0 Hit Points
- **问题发现**: 当前 `IsDead()` 和 `IsUnconscious()` 实现不符合规则

#### 规则要点

| 规则 | 描述 |
|------|------|
| HP = 0 | 陷入昏迷（Unconscious），而非死亡 |
| 死亡豁免 | 每回合开始时进行 DC 10 豁免 |
| 成功 3 次 | 稳定状态（Stable），不再需要豁免 |
| 失败 3 次 | 死亡 |
| 受到伤害 | 豁免失败 +1，暴击则失败 +2 |
| 受到 >= MaxHP 伤害 | 立即死亡 |
| 受到治疗 | HP 恢复，脱离昏迷 |

#### 实现范围

**数据层**:
- 模型: 扩展 `HP` 结构体，添加状态字段
  ```go
  type HPState string
  const (
      HPStateNormal     HPState = "normal"      // 正常
      HPStateUnconscious HPState = "unconscious" // 昏迷
      HPStateStable     HPState = "stable"      // 稳定
      HPStateDead       HPState = "dead"        // 死亡
  )
  ```
- 存储: 无需修改（状态由 DeathSaves 推导）

**业务层**:
- 服务: `CharacterService` 添加死亡豁免处理方法
- 规则: `rules/death_save.go` - 死亡豁免规则实现
  - `IsUnconscious()` - HP = 0 且未稳定
  - `IsDead()` - DeathSaves.Failures >= 3 或 HP < -MaxHP
  - `IsStable()` - DeathSaves.Successes >= 3
  - `MakeDeathSave()` - 执行死亡豁免
  - `TakeDamageWhileUnconscious()` - 昏迷时受伤处理

**接口层**:
- 端点: 扩展 `update_character` 支持死亡豁免操作

**测试层**:
- 单元测试: 所有状态判断、豁免逻辑
- 集成测试: 完整死亡豁免流程

#### 约束条件

- 前置任务: T3-6（需要 DeathSaves 结构体）
- 技术约束: 必须严格遵循 PHB 规则
- 边界条件:
  - HP = 0 与 HP < 0 的区别
  - 暴击伤害在昏迷状态的特殊处理
  - 稳定后受伤害重置状态

#### 验收标准

- [ ] 实现完成: 所有死亡相关规则方法
- [ ] 测试通过: 单元测试覆盖所有边界条件

### Task T3-8: 负重规则

**一句话描述**: 实现负重计算和限制规则（可选功能）。

#### 需求来源

- **规则书**: PHB 第7章 - Using Ability Scores / Lifting and Carrying
- **问题发现**: 装备系统有重量字段，但未实现负重计算

#### 规则要点

| 规则 | 描述 |
|------|------|
| 负重能力 | 力量值 × 15（磅） |
| 推/拖/举起 | 力量值 × 30（磅） |
| 超重惩罚 | 速度降至 5 尺（如启用变体规则） |
| 体型修正 | 小型生物负重 × 0.5，大型 × 2 |

> **注意**: 负重规则为可选功能，非核心规则。默认实现简化版本。

#### 实现范围

**数据层**:
- 模型: 新增 `Encumbrance` 结构体
  ```go
  type Encumbrance struct {
      Carried      int   `json:"carried"`       // 当前负重（磅）
      Capacity     int   `json:"capacity"`      // 负重能力
      PushDragLift int   `json:"push_drag_lift"` // 推/拖/举起
      IsEncumbered bool  `json:"is_encumbered"` // 是否超重
  }
  ```
- 存储: 无需修改（计算值，不持久化）

**业务层**:
- 服务: `CharacterService` 添加负重计算方法
- 规则: `rules/encumbrance.go` - 负重规则实现
  - `CalculateCapacity(str int, size Size) int` - 计算负重能力
  - `CalculateCarried(equipment []EquipmentItem, inventory []InventoryItem) int` - 计算当前负重
  - `IsEncumbered(carried, capacity int) bool` - 判断是否超重

**接口层**:
- 端点: `get_character` 响应包含负重信息

**测试层**:
- 单元测试: 负重计算、体型修正
- 集成测试: 装备变更后负重更新

#### 约束条件

- 前置任务: T3-6（需要 EquipmentItem、InventoryItem）
- 技术约束: 可选功能，可配置开关
- 边界条件: 体型修正、负数重量过滤
